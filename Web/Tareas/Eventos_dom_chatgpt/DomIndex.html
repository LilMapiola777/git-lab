Preguntas

  1. ¿Qué es el DOM?

    El DOM (Document Object Model) es una representación estructurada del documento HTML en forma de árbol. Cada parte del HTML (etiquetas, texto, atributos, etc.) se convierte en un nodo que puede ser accedido y manipulado con JavaScript.

    Fuente: MDN Web Docs - DOM

    https://developer.mozilla.org/es/docs/Web/API/Document_Object_Model

  2. ¿Qué es un nodo?

    Un nodo es cualquier unidad del DOM. Puede ser un elemento HTML, un texto, un atributo, un comentario, etc. Toda la página está compuesta por nodos interconectados jerárquicamente.

    Fuente: MDN Web Docs - Node

    https://developer.mozilla.org/es/docs/Web/API/Node

  3. ¿Cómo se accede a un elemento del DOM?

    Se accede usando métodos del objeto document, como getElementById, querySelector, getElementsByTagName, entre otros. Estos permiten seleccionar y manipular elementos específicos del HTML.

    Fuente: MDN Web Docs - Document

    https://developer.mozilla.org/es/docs/Web/API/Document

  4. ¿Cómo se modifica un elemento del DOM?

    Una vez accedido un elemento, se puede modificar su contenido, estilos, atributos o clases mediante propiedades como innerText, innerHTML, style, entre otras.

    Fuente: MDN Web Docs - Element

    https://developer.mozilla.org/es/docs/Web/API/Element

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    Los métodos getElementBy* son específicos (por ID, nombre, clase, etc.), mientras que querySelector usa selectores CSS, permitiendo búsquedas más precisas y flexibles. Además, querySelectorAll devuelve una lista estática.

    Fuente: MDN Web Docs - querySelector

    https://developer.mozilla.org/es/docs/Web/API/Document/querySelector

  6. ¿Qué es un evento?

    Un evento es una acción que ocurre en la página (clic, presionar tecla, cargar imagen, mover el mouse, etc.) y que puede ser detectada y manejada con JavaScript para ejecutar código.

    Fuente: MDN Web Docs - Eventos

    https://developer.mozilla.org/es/docs/Learn/JavaScript/Building_blocks/Events

  7. ¿Qué es un listener?

    Un listener es una función que "escucha" un evento específico. Cuando el evento ocurre, se ejecuta el código que se haya definido para ese evento mediante addEventListener.

    Fuente: MDN Web Docs - addEventListener

    https://developer.mozilla.org/es/docs/Web/API/EventTarget/addEventListener

  8. ¿Qué es un callback?

    Un callback es una función que se pasa como argumento a otra función. Se ejecuta cuando ocurre una acción específica. Es común en eventos, temporizadores, peticiones asíncronas, etc.

    Fuente: MDN Web Docs - Callback function

    https://developer.mozilla.org/es/docs/Glossary/Callback_function

  9. ¿Qué es un objeto de evento?

    Es un objeto que se crea automáticamente cuando ocurre un evento. Contiene información sobre el evento, como qué lo causó, en qué elemento ocurrió y otros detalles como coordenadas, teclas presionadas, etc.

    Fuente: MDN Web Docs - Event

    https://developer.mozilla.org/es/docs/Web/API/Event

Conclusiones

Experiencia general

Trabajar con ChatGPT fue muy rápido y sencillo. Siento que es similar a tener un profesional en tu equipo, aunque no te da todo correctamente a la primera vez. Normalmente te da los códigos con buena funcionalidad, sin embargo vienen con algunos bugs que se deben de resolver volviendo a hacer una consulta.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Usaría ChatGPT cuando quiero entender algo de manera profunda y cuando tengo problemas con algún bug en mi código
Usaría Copilot cuando estoy programando y necesito generar algunas líneas o datos prueba para mi código.

Ventajas de usar AI para programar

  • Hace el trabajo más rápido
  • Depende de la manera en que lo uses, puedes aprender mucho al programar
  • No gastar tanto tiempo buscando errores
  • Te acostumbra a tener una buena estrucutra en tu código

Desventajas de usar AI para programar

  • Puede darte código incorrecto
  • Se puede generar una dependencia a el
  • Puedes copiar y no es una buena práctica
  • Si lo usas sin analizar y pedir explicación de lo que pides, no aprendes
